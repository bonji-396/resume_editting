履歴書 Editorの詳細

概要

履歴書を作成するページです

完全にオリジナル作品となります。

履歴書作成ウェブアプリです。 HTML、CSS（SCSS）、JavaScriptのみで構成されています。フロントエンドで終息しサーバに（個人情報など）送信することはありません。

また、編集中データ自動保存していますが、プラウザを閉じれば個人情報が残ることもありません。共用端末での利用でも安心してお使いいただけます。

継続利用ができるようにJSON形式テキストファイルでインポート／エクスポートが可能です。

※ ただし、個人情報を扱うものですので、インポート、HTML出力、印刷・PDF印刷したデータの管理はご自身でしっかり行って下さい。

仕様

初期仕様です。幾つか実装を変更にて対応してます。

目的

就職活動支援サイトでの履歴書作成支援で、EXCELなど利用できない環境でも履歴書を作成することができるように支援するためのページ

仕様詳細

  • 初期画面は編集モード（contenteditable=true)として、空欄の履歴書を表示
  • 見た目のレイアウトは一般の履歴書と同じ
  • 印刷及び印刷プレビューでは、A4、B5サイズので印刷用スタイルシートを充てて表示する
  • 写真はドラックアンドドロップ、＋ボタン押下でファイルシステムから選択で挿入可能
  • データはSessionStrageに自動保存
  • JSON型データへシリアライズ保存

特徴

  • 履歴書（JIS規格）見た目のままで編集できる
  • A4、A3見開き、B5、B4見開きのそれぞれを選択が可能
  • HTMLファイル出力が可能
  • 再編集用にJSON形式で保存・読み込みが可能
  • 編集内容は自動保存してくれる

履歴書（JIS規格）見た目のままで編集できる

今時この規格で作ること自体？って感じではありますが、使い慣れたJIS規格系の形式で編集できます。

苦労した点

項目
内容
Data URLと、Blob URLの違い。さらにBASE64の存在
多分、ウェブ業界では当たり前の前提知識だと思うんですが、自身の知識不足でこういったデータの扱いと学習に時間がかかりました。
input type="file"
基本的にブラウザはサーバーに接続してリソースを持ってきたり、送ったりを前提として動きます。なので、ローカルのリソースを使う場合は基本的にはサーバへ送信することが主となります。
まずはinput要素とその属性fileを利用してファイルを読み込むのがセオリーのようです。ここら辺も想定したものとは違い、体得するのに若干時間がかかりました。
CSSで複雑なレイアウトと、その各定義した値が用紙毎に変更すること
当初は純粋なCSSで作業を進めました。ですが、複雑なレイアウトと、その各定義した値が用紙毎に変更することになるため、作業をしていて心が折れそうになりました。 一旦、SASS（SCSS）を学び、再び挑戦したのですが、これの判断は正解だと思います。

反省点と今後

反省点

入力抑止等の制限処理をしていない。こちらはサーバサイドに送信することがない仕様なので、安全面で問題ないと判断しそのままです。

実装にあたっては、input要素を利用した方がブラウザに頼る分断然楽に実装できます。

ですが、HTML出力を想定していたので、HTMLとCSSを基準にするとcontentEditableの利用が楽だと判断しました。

ただ今考えると、ここまでくると最初からJavascriptオンリーで全ての要素を生成するような方向で実装すれば、contentEditableに拘らなくてよかったのかもしれません。

そしてその場合はinput要素を生成し入力抑止の部分はブラウザ任せで実装できたと思います。

今後

早速、スマートフォン対応したいと思います。

  • スマートフォン用メニューの実装
  • 入力編集方法の変更を検討
